Metodos POST y GET para enviar datos.

GET

  • se utiliza para solicitar datos del servidor.
  • los datos del formulario se agregan a la URL como parametros de consulta
  • es visible en la variable de direccion del navegador yh puede ser marcado o guardado en el navegador
  • es adecuado para solicitudes que no cambian el estado del servidor (consulta de bsase de datos y busqueda)
  • los datos tienen una longitud limitada por las restricciones de longitud URL

Metodos POST y GET para enviar datos.

POST

  • envia datos al servidor para procesarlo datos del servidor.
  • los datos del formulario son visibles
  • es adecuado para enviar datos confidenciales y ademas pueden ser mas extensos
  • no se utiliza para solicitudes que cambian el estado de un servidor de ser asi debe usarse con precaucion
  • no tiene restricciones de longitud como el metodo GET
